# Work Log — 25 June 2026

## Bugs Addressed

### Bug 1 — RFQ/Quotation forms not pre-filling from opportunity
**Status:** Fixed (pending final verification after root-cause fix)  
**Goal:** When a user opens `Create RFQ` or `Direct Quotation` from an Opportunity detail page, the form should pre-fill customer, contact, product, and assigned user using the `opportunityId` query parameter.

**Root cause (resolved):** The new unified context API `GET /api/opportunities/[id]/context` was ordering `opportunityContacts` by `createdAt`, but the `OpportunityContact` model has `addedAt`, not `createdAt`. This caused a Prisma validation error and the API returned an error, so the frontend fell back to manual selection.

**Files changed:**
- `app/api/opportunities/[id]/context/route.ts` — created; returns opportunity context including customer, contacts, assigned user, linked RFQ; added top-level error logging.
- `app/api/opportunities/[id]/quotation-context/route.ts` — fixed same `orderBy` issue (`createdAt` → `addedAt`).
- `app/(dashboard)/rfq/new/page.tsx` — fetches context, pre-fills customer, contact, product, assigned user; shows loading + error notice; navigates back to opportunity after save.
- `app/(dashboard)/quotations/new/page.tsx` — fetches context, pre-fills customer, contact, assigned user, first line-item product; customer is editable; shows loading + error notice.
- `prisma/schema.prisma` — added `assignedUser` / `assignedUserId` to `Quotation` model.
- `app/api/quotations/route.ts` — stores `assignedUserId` on quotation create.

**Verification:**
- `npx tsc --noEmit` passes.
- Refresh the RFQ/Quotation page launched from an opportunity; customer/contact/assigned user should now populate.

### Bug 4 — Product Catalogue categories not loading / adding
**Status:** Fixed  
**Goal:** Make the Product Catalogue > Categories module functional so categories can be listed and added.

**Root cause:** The categories API `GET /api/catalogue/categories` was building a Prisma `where` clause with `deletedAt: null`. The `ProductCategory` model does not have a `deletedAt` field (only `Product` has soft-delete columns), so every GET request failed with a Prisma validation error and the page showed "No categories found".

**Files changed:**
- `app/api/catalogue/categories/route.ts` — removed the invalid `deletedAt: null` filter; added `search` query handling so the UI search box works.

**Impact:**
- `app/(dashboard)/catalogue/categories/page.tsx` — categories now load correctly.
- `app/(dashboard)/catalogue/products/page.tsx` — category filter dropdown now populates.
- `app/(dashboard)/catalogue/products/new/page.tsx` — category selector now populates.
- `app/(dashboard)/catalogue/products/[id]/page.tsx` — category selector now populates.

**Verification:**
- Refresh `/catalogue/categories`; the empty state should appear only when no categories exist, and clicking **Add Category** should create a category successfully.
- `/catalogue/products/new` should show categories in the Category dropdown.
